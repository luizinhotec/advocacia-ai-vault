# Checklist de Pré-Publicação

Execute este checklist para **todo** conteúdo antes de publicar. Não pule etapas.

---

## A — Qualquer tipo de conteúdo

- [ ] Aprovação escrita do escritório obtida e registrada?
- [ ] Conteúdo alinhado com o pilar editorial declarado (ver [[estrategia-editorial]])?
- [ ] Nenhum dado pessoal de cliente presente (nome, CPF, telefone, caso)?
- [ ] Nenhuma promessa de resultado?
- [ ] Nenhuma comparação com outros escritórios ou advogados?
- [ ] Tom sóbrio e profissional — nada sensacionalista?
- [ ] Redirecionamento para consulta presente (quando aplicável)?

---

## B — Post de rede social (imagem + legenda)

- [ ] Identidade visual seguida (paleta, tipografia, logo) — ver [[identidade-visual]]?
- [ ] Legenda revisada: sem jargão excessivo, sem orientação jurídica concreta?
- [ ] Hashtags aprovadas para o canal?
- [ ] Imagem sem rostos de clientes ou documentos identificáveis?
- [ ] Caption não usa honorário como atrativo?
- [ ] Formato correto para o canal (dimensões, proporção)?

---

## C — Vídeo com avatar

- [ ] Roteiro aprovado antes da produção — ver [[roteiro-padrao-video]]?
- [ ] Vídeo aprovado pelo titular da voz (advogado(a)) — ver [[consentimento-voz-clonada]]?
- [ ] Avatar identificado como IA no vídeo (legenda ou watermark)?
- [ ] Nenhuma fala que simule orientação jurídica personalizada?
- [ ] Logo e identidade visual presentes conforme [[identidade-visual]]?
- [ ] Qualidade de áudio e vídeo adequada para publicação profissional?
- [ ] CTA final não usa honorário como atrativo?
- [ ] Arquivo registrado em [[assets-de-midia]]?

---

## D — Atualização do agente de atendimento

- [ ] Nota compilada a partir de fonte em `raw/`?
- [ ] Frontmatter YAML completo e válido?
- [ ] Links `[[...]]` apontando para notas existentes?
- [ ] Lint do SCHEMA.md executado (seção 3)?
- [ ] Nota aprovada pelo escritório e status atualizado para `revisado`?
- [ ] Nenhuma orientação jurídica concreta na nota?
- [ ] `_index.md` da camada pública atualizado se for nota nova?

---

## Registro

Após publicar, registrar na planilha de controle:
- Data de publicação
- Canal
- Título/descrição do conteúdo
- Tipo (post / vídeo / atualização de vault)
- Aprovado por
- Link do conteúdo publicado

Ver [[assets-de-midia]] para localização da planilha.
